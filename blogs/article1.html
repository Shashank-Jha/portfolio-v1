difference between "==" & "===" in javascript

Both == and === operators are equality operators in javascript. The main diffrence between both is :

  • Before comparision, the == operator changes the given variables in a same type and compares the values.
  • While, the === operator strictly compares the variables types as well as values.

  • When we compared var 'a' with var 'b' using '==', then javascript compares both the values and as the value of both variables are 10, the output is "True".
  • Now, when we again compare var 'a' with var 'b' using '===', then javascript compared both the variable's values as well as their data-type and as the value & type of both variables are 10, the output is "True".
Now, let's see what happened when we compared var 'a' with var 'c'.
  • In this case, when we compare var 'a' with var 'c' using '==' operator, though the type of var 'c' is of a string but '==' compares only the value and the output is "true".
  • But, in this case when we compare var 'a' with var 'c' using '===' operator, the operator checks and compares both the value as well as the type and hence as var 'a' is of type integer and var 'c' is of type string, the output is "False".

Conclusion

I hope this little effort of mine to explain the difference between '==' and '===' operator in javascript made you understand the difference well.
So to conclude, both the operators are equality check operators but '===' is strict-equality operator.
When we use '==' operator it only compares the values, on the other hand "===" operator compares value as well as the type of the given data.
